What I've Tried and Lesson Learned Throughout Making This Website and opinions about it

  • many tools for prototyping your website such as (what I've tried) Invision Studio, Justinmind, etc cannot export your design to a full HTML files (at least not freely). the best way to make a website is by it from scratch.
  • making a website from scratch can be a daunting task, so it would be better if not faster to make use of existing libraries and frameworks to maximize production. I've explored several libraries and interested in Bootstrap as a huge time saver to making components such as buttons and etc and Animate.css to add a little bit of animation without having to do a lot of work.
  • designing with CSS can be a pain in the back. because even if you had live server in vs code, still it would be very hard to know why something doesn't work when I code this way. so it's nice to have a CSS Debugger like Chrome DevTools that lets you debug your CSS and design your web realtime. just dont forget to save the changes in the actual css file
  • hosting your website can be bothersome. you either need a server to do that or be hosted by someone else.some of those hosting sites are free such as GitHub pages and Netlify and that's what i'm using.
